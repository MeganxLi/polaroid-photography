# 專案需求分析 (Polaroidly)

根據提供的 Wireframe UI 圖片，整理出以下前端網頁需求與功能規格：

## 1. 視覺風格與版面佈局 (UI/UX & Layout)
- **整體風格**：採用柔和、活潑的粉彩色調（Pastel Colors），帶有可愛、復古的質感。
- **背景設計**：乾淨的淺色背景，並散落點綴幾顆帶有粉色邊框的「四角閃爍星星」圖案（可加入緩慢浮動或閃爍的微動畫）。
- **網頁佈局**：單頁式應用程式（SPA），重要元素分佈於四周，中央留給核心的「相機與相片」互動區。

## 2. 導覽列與文字元素 (Typography & Navigation)
- **左上角 Logo**：「Polaroidly」品牌名稱，搭配四角星星圖示，字體顏色使用珊瑚粉/粉紅色。
- **右上角按鈕**：「[⚡] 操作說明」按鈕(battery-charging icon)。點擊後預期會展開 Modal 彈出視窗或提示框，引導使用者如何操作（拍照、濾鏡、文字、下載等）。
- **右下角外部連結**：顯示 GitHub 圖示與名稱「Megna」，點擊應另開分頁前往開發者的 GitHub 頁面。

## 3. 核心互動區：拍立得相機與相片 (Core Feature: Camera & Photo)
- **相機主體**：
  - 畫面正中央有一台立體的復古相機短插畫 / SVG 組件。
  - 左側帶有粉紅、橘黃、淺綠色相間的垂直復古條紋。
  - 具備鏡頭、觀景窗、紅點（閃光燈標示）等細節。
- **相片產出區域**：
  - 相機底部的暗槽處會「吐出」一張拍立得相片（預期應具備相片向下滑出的動畫效果）。
  - **影像區塊**：使用者可上傳或拖曳圖片至此區域（Mockup 範例為一隻熟睡的白貓）。
  - **文字輸入區塊**：拍立得相片的下方白邊區域保留了文字空間（Mockup 中顯示「Meow~」），允許使用者點擊此處並自訂想留下的文字/日期。

## 4. 濾鏡/色彩主題選擇器 (Filter Selector)
- **位置與標題**：位於畫面左下方，標題顯示「選擇濾鏡」。
- **選項**：提供四個圓形色塊按鈕（粉紅、橘黃、淺綠、薄荷藍）。
- **互動反饋**：
  - 目前被選取的濾鏡（圖中為粉紅色）其外圍會有發光/外框效果，以突顯選取狀態。
  - 當使用者切換選擇不同的色塊時，預期會改變右邊紅色小圓點的顏色。

## 5. 預期的隱藏需求 (Expected Implicit Requirements)
- **拍照機制**：點擊拍攝按鈕後，開啟鏡頭可以攝影，並將相片吐出。
- **下載/匯出機制**：完成相片上傳、套用濾鏡與輸入文字後，應該有點擊「儲存/下載相片」將結果繪製並匯出為 PNG/JPEG 檔的功能。
- **RWD 響應式設計**：雖然提供的是電腦版比例，但這類服務應確保在手機裝置上也能順暢操作（例如濾鏡移到相機正下方、元件縮放等）。
